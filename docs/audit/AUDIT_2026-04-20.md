# System Audit — 2026-04-20

Post-Phase-5 cleanup pass. Findings from automated sweeps (Phase A) and
targeted review (Phase B). Each finding has severity, location, and a
proposed fix. Ordered by severity within each section.

**Severity key:**
- **CRIT** — active bug, can crash/lose data/produce wrong behavior
- **HIGH** — real correctness risk even if not currently firing
- **MED** — quality issue, maintenance burden, or latent bug
- **LOW** — cosmetic, style, doc drift

---

## 1. CRITICAL — active or imminent correctness bugs

### 1.1 Undefined `time` in retail_sector_screener.py
**Locations:** `agents/retail_sector_screener.py:334, 340, 387, 393`
**From:** ruff F821
**Risk:** `time.sleep()` (or similar) called without `import time`. If those
code paths are ever reached, **NameError crash**. May be rarely-exercised
circuit-breaker or retry code, which is exactly where we least want a
silent failure mode.
**Fix:** Add `import time` at top of file. Verify the 4 call sites.

### 1.2 Undefined `timezone` in retail_trade_logic_agent.py
**Locations:** `agents/retail_trade_logic_agent.py:782, 3052`
**From:** ruff F821
**Risk:** `timezone` referenced without import. Same NameError risk —
likely in a datetime branch that's rarely hit.
**Fix:** `from datetime import timezone` at top, or fix references if
typo'd.

### 1.3 Undefined `trade_events` in trader
**Location:** `agents/retail_trade_logic_agent.py:2645`
**From:** ruff F821
**Risk:** Variable used before assignment in some branch. Could crash a
per-signal evaluation if hit.
**Fix:** Trace scope, initialize `trade_events = 0` at function start.

### 1.4 Candidate Generator dedup broken (known)
**Location:** `src/retail_database.py::add_candidate_signal`
**Symptom:** Running candidate_generator twice emits the same ticker as
new signals instead of updating. Observed: 7 emitted → 7 emitted (not 7
updated).
**Consequence:** This is half of the AAPL runaway root cause. Every 30
min, same ticker → new signal ID → new window → new entry.
**Fix:** Investigate the `tx_date=today` match — likely date format or
timezone mismatch between INSERT and SELECT. Probably INSERT doesn't set
`tx_date` column (defaults NULL) while the SELECT matches on `tx_date`.

### 1.5 Gate 4 has no ticker-level dedup (known)
**Location:** `agents/retail_trade_logic_agent.py::gate4_eligibility`
**Symptom:** Gate 4 checks sector concentration (<3 per sector) but not
"already hold this ticker." Combined with 1.4, this is why AAPL stacked
3 positions per customer.
**Consequence:** The AAPL runaway.
**Fix:** Add `4_TICKER_DEDUP` sub-gate: if `ticker in {p['ticker'] for p
in positions if p['status']=='OPEN'}` → SKIP.

### 1.6 Redundant if-condition blocks (dead branches)
**Locations:** `agents/retail_trade_logic_agent.py:3149, 3178, 3255`
**From:** vulture (100% confidence, 12 / 35 / **265 lines**)
**Risk:** Large dead blocks mean the logic the reader thinks is live
isn't. The 265-line one is probably `if True: ...` (Gate 0 passthrough)
which is harmless but the 12 and 35 line ones are suspicious — they may
indicate logic that never runs in production.
**Fix:** Read each block, confirm dead vs. always-true, delete or
restructure.
